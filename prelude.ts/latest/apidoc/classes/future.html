Class Future<T>

A Future is the equivalent, and ultimately wraps, a javascript Promise. A fundamental difference is that Futures are lazy. Just defining a Future won't trigger its execution, you'll have to use it for that (call Future.map, Future.flatMap, use await on it and so on). This means that a Future can have one of the four states:

  1. not triggered
  2. pending
  3. fulfilled
  4. rejected

That first state doesn't exist with Javascript Promises. In addition, while Futures support the Future.then call (so that among others you can use await on them), you should call Future.map and Future.flatMap.

Futures represent an asynchronous computation. A Future will only ever be computed once at most. Once it's computed, calling Future.map or await will return instantly.
